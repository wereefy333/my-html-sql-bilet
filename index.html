Билет 1

Вопрос

Создайте таблицу "Студенты" с полями: id (первичный ключ), имя, фамилия, курс.
Вывести всех студентов 2-го курса.

Ответ

CREATE TABLE Студенты (
    id INT PRIMARY KEY,
    имя VARCHAR(50),
    фамилия VARCHAR(50),
    курс INT
);

SELECT * FROM Студенты WHERE курс = 2;
Правильное заполнение — нажмите для примера 
INSERT INTO Студенты (id, имя, фамилия, курс) VALUES
(1, 'Иван', 'Иванов', 2),
(2, 'Мария', 'Петрова', 1),
(3, 'Алексей', 'Сидоров', 2),
(4, 'Ольга', 'Кузнецова', 3);
Пояснение:
  • PRIMARY KEY (id) — нужен для уникальной идентификации каждого студента, чтобы не было дубликатов.
  • VARCHAR для имени и фамилии — позволяет хранить текстовые значения разной длины.
  • INT для курса — так как курс это целое число.
  • Запрос SELECT ... WHERE курс = 2 выбирает только студентов второго курса.

Билет 3

Вопрос

Создайте таблицу "Товары" (id, название, цена).
Вывести 3 самых дорогих товара.

Ответ

CREATE TABLE Товары (
    id INT PRIMARY KEY,
    название VARCHAR(100),
    цена DECIMAL(10,2)
);

SELECT * FROM Товары ORDER BY цена DESC LIMIT 3;
Правильное заполнение — нажмите для примера 
INSERT INTO Товары (id, название, цена) VALUES
(1, 'Ноутбук', 80000.00),
(2, 'Смартфон', 50000.00),
(3, 'Планшет', 30000.00),
(4, 'Монитор', 20000.00);
Пояснение:
  • PRIMARY KEY (id) — уникальный идентификатор для каждого товара.
  • DECIMAL для цены — позволяет хранить дробные значения (например, 99.99).
  • Сортировка по убыванию (ORDER BY ... DESC) и LIMIT 3 позволяют выбрать только три самых дорогих товара.

Билет 5

Вопрос

Создайте таблицы "Группы" (id, название) и "Студенты" (id, имя, id_группы).
Вывести студентов определенной группы.

Ответ

CREATE TABLE Группы (
    id INT PRIMARY KEY,
    название VARCHAR(50)
);
CREATE TABLE Студенты (
    id INT PRIMARY KEY,
    имя VARCHAR(50),
    id_группы INT,
    FOREIGN KEY (id_группы) REFERENCES Группы(id)
);

SELECT * FROM Студенты WHERE id_группы = 1;
-- Или по названию группы:
SELECT Студенты.* FROM Студенты JOIN Группы ON Студенты.id_группы = Группы.id WHERE Группы.название = 'Название_группы';
Правильное заполнение — нажмите для примера 
INSERT INTO Группы (id, название) VALUES
(1, 'ПМИ-101'),
(2, 'ПМИ-102');

INSERT INTO Студенты (id, имя, id_группы) VALUES
(1, 'Иван', 1),
(2, 'Мария', 2),
(3, 'Ольга', 1);
Пояснение:
  • PRIMARY KEY в обеих таблицах — для уникальности записей.
  • FOREIGN KEY (id_группы) — обеспечивает связь между студентом и группой, поддерживает целостность данных.
  • Для поиска студентов по группе можно использовать либо id_группы, либо JOIN по названию группы.

Билет 6

Вопрос

Создайте таблицу "Заказы" (id, дата, сумма).
Вывести заказы за последние 7 дней.

Ответ

CREATE TABLE Заказы (
    id INT PRIMARY KEY,
    дата DATE,
    сумма DECIMAL(10,2)
);

SELECT * FROM Заказы
WHERE дата >= CURRENT_DATE - INTERVAL '7 day';
Правильное заполнение — нажмите для примера 
INSERT INTO Заказы (id, дата, сумма) VALUES
(1, '2024-05-01', 1500.00),
(2, '2024-05-05', 2500.00),
(3, '2024-05-07', 3200.00);
Пояснение:
  • PRIMARY KEY (id) — уникальный идентификатор заказа.
  • DATE для даты — позволяет хранить дату заказа.
  • DECIMAL для суммы — для хранения денежных значений с копейками.
  • Фильтрация по дате с помощью WHERE дата >= CURRENT_DATE - INTERVAL '7 day' позволяет выбрать только заказы за последние 7 дней.

Пояснение:
  • PRIMARY KEY (id) — уникальный идентификатор продукта.
  • VARCHAR для текстовых полей.
  • GROUP BY используется для группировки по категории, а COUNT(*) — для подсчёта количества товаров в каждой категории.

Билет 11

Вопрос

Создайте таблицу "Студенты" (id, имя, балл).
Вывести студентов с баллом выше среднего.

Ответ

    id INT PRIMARY KEY,

(1, 'Иван', 4.5),
(2, 'Мария', 3.8),
(3, 'Ольга', 4.9);
Пояснение:
  • PRIMARY KEY (id) — уникальный идентификатор студента.
  • DECIMAL для балла — позволяет хранить дробные значения.
  • Вложенный SELECT AVG(балл) вычисляет средний балл, а основной запрос выбирает студентов с баллом выше среднего.

Билет 13

Вопрос

Создайте таблицы "Клиенты" (id, имя) и "Заказы" (id, сумма, id_клиента).
Вывести общую сумму заказов каждого клиента.

Ответ

CREATE TABLE Клиенты (
    id INT PRIMARY KEY,
    имя VARCHAR(100)
);
CREATE TABLE Заказы (
    id INT PRIMARY KEY,
    сумма DECIMAL(10,2),
    id_клиента INT,
    FOREIGN KEY (id_клиента) REFERENCES Клиенты(id)
);

SELECT Клиенты.имя, SUM(Заказы.сумма) AS Общая_сумма
FROM Клиенты
LEFT JOIN Заказы ON Клиенты.id = Заказы.id_клиента
GROUP BY Клиенты.имя;
Правильное заполнение — нажмите для примера 
INSERT INTO Клиенты (id, имя) VALUES
(1, 'Иван'),
(2, 'Мария');

INSERT INTO Заказы (id, сумма, id_клиента) VALUES
(1, 1000.00, 1),
(2, 2000.00, 1),
(3, 1500.00, 2);
Пояснение:
  • PRIMARY KEY в обеих таблицах — для уникальности записей.
  • FOREIGN KEY (id_клиента) — обеспечивает связь заказа с клиентом.
  • LEFT JOIN позволяет получить всех клиентов, даже если у них нет заказов.
  • SUM() и GROUP BY используются для подсчёта общей суммы заказов по каждому клиенту.

CREATE TABLE Товары (
    id INT PRIMARY KEY,
    название VARCHAR(100),
    id_категории INT,
    FOREIGN KEY (id_категории) REFERENCES Категории(id)
);

SELECT Категории.название, COUNT(Товары.id) AS Количество_товаров
FROM Категории
LEFT JOIN Товары ON Категории.id = Товары.id_категории
GROUP BY Категории.название;
Правильное заполнение — нажмите для примера 
INSERT INTO Категории (id, название) VALUES
(1, 'Молочные'),
(2, 'Выпечка');

INSERT INTO Товары (id, название, id_категории) VALUES
(1, 'Молоко', 1),
(2, 'Хлеб', 2);
Пояснение:
  • PRIMARY KEY в обеих таблицах — для уникальности записей.
  • FOREIGN KEY (id_категории) — обеспечивает связь товара с категорией.
  • LEFT JOIN позволяет получить все категории, даже если в них нет товаров.
  • COUNT() и GROUP BY используются для подсчёта количества товаров в каждой категории.

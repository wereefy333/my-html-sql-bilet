Билет 1

Вопрос

Создайте таблицу "Студенты" с полями: id (первичный ключ), имя, фамилия, курс.
Вывести всех студентов 2-го курса.

Ответ

CREATE TABLE Студенты (
    id INT PRIMARY KEY,
    имя VARCHAR(50),
    фамилия VARCHAR(50),
    курс INT
);

SELECT * FROM Студенты WHERE курс = 2;
Пояснение:
  • PRIMARY KEY (id) — нужен для уникальной идентификации каждого студента, чтобы не было дубликатов.
  • VARCHAR для имени и фамилии — позволяет хранить текстовые значения разной длины.
  • INT для курса — так как курс это целое число.
  • Запрос SELECT ... WHERE курс = 2 выбирает только студентов второго курса.

Билет 3

Вопрос

Создайте таблицу "Товары" (id, название, цена).
Вывести 3 самых дорогих товара.

Ответ

CREATE TABLE Товары (
    id INT PRIMARY KEY,
    название VARCHAR(100),
    цена DECIMAL(10,2)
);

SELECT * FROM Товары ORDER BY цена DESC LIMIT 3;
Пояснение:
  • PRIMARY KEY (id) — уникальный идентификатор для каждого товара.
  • DECIMAL для цены — позволяет хранить дробные значения (например, 99.99).
  • Сортировка по убыванию (ORDER BY ... DESC) и LIMIT 3 позволяют выбрать только три самых дорогих товара.

Билет 5

Вопрос

Создайте таблицы "Группы" (id, название) и "Студенты" (id, имя, id_группы).
Вывести студентов определенной группы.

Ответ

CREATE TABLE Группы (
    id INT PRIMARY KEY,
    название VARCHAR(50)
);
CREATE TABLE Студенты (
    id INT PRIMARY KEY,
    имя VARCHAR(50),
    id_группы INT,
    FOREIGN KEY (id_группы) REFERENCES Группы(id)
);

SELECT * FROM Студенты WHERE id_группы = 1;
-- Или по названию группы:
SELECT Студенты.* FROM Студенты JOIN Группы ON Студенты.id_группы = Группы.id WHERE Группы.название = 'Название_группы';
Пояснение:
  • PRIMARY KEY в обеих таблицах — для уникальности записей.
  • FOREIGN KEY (id_группы) — обеспечивает связь между студентом и группой, поддерживает целостность данных.
  • Для поиска студентов по группе можно использовать либо id_группы, либо JOIN по названию группы.

Вопрос

Ответ

CREATE TABLE Заказы (
    id INT PRIMARY KEY,
    дата DATE,
    сумма DECIMAL(10,2)
);

WHERE дата >= CURRENT_DATE - INTERVAL '7 day';
Пояснение:

  • DATE для даты — позволяет хранить дату заказа.
  • DECIMAL для суммы — для хранения денежных значений с копейками.
  • Фильтрация по дате с помощью WHERE дата >= CURRENT_DATE - INTERVAL '7 day' позволяет выбрать только заказы за последние 7 дней.

Билет 13

Вопрос

Создайте таблицы "Клиенты" (id, имя) и "Заказы" (id, сумма, id_клиента).
Вывести общую сумму заказов каждого клиента.

Ответ

CREATE TABLE Клиенты (
    id INT PRIMARY KEY,
    имя VARCHAR(100)
);
CREATE TABLE Заказы (
    id INT PRIMARY KEY,
    сумма DECIMAL(10,2),
    id_клиента INT,
    FOREIGN KEY (id_клиента) REFERENCES Клиенты(id)
);

SELECT Клиенты.имя, SUM(Заказы.сумма) AS Общая_сумма
FROM Клиенты
LEFT JOIN Заказы ON Клиенты.id = Заказы.id_клиента
GROUP BY Клиенты.имя;
Пояснение:
  • PRIMARY KEY в обеих таблицах — для уникальности записей.
  • FOREIGN KEY (id_клиента) — обеспечивает связь заказа с клиентом.
  • LEFT JOIN позволяет получить всех клиентов, даже если у них нет заказов.
  • SUM() и GROUP BY используются для подсчёта общей суммы заказов по каждому клиенту.

SELECT группа, COUNT(*) AS Количество_студентов
FROM Студенты
GROUP BY группа;
Пояснение:
  • PRIMARY KEY (id) — уникальный идентификатор студента.
  • GROUP BY используется для группировки по группе, а COUNT(*) — для подсчёта количества студентов в каждой группе.

Вопрос

Создайте таблицу "Продукты" (id, название, цена).
Вывести самый дорогой продукт.

Ответ

CREATE TABLE Продукты (
    id INT PRIMARY KEY,
    название VARCHAR(100),
    цена DECIMAL(10,2)
);

SELECT * FROM Продукты
ORDER BY цена DESC
LIMIT 1;
Пояснение:
  • PRIMARY KEY (id) — уникальный идентификатор продукта.
  • ORDER BY ... DESC LIMIT 1 позволяет выбрать только самый дорогой продукт.
